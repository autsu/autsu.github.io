私有镜像仓库 harbor 部署实践

前言

参考官方文档 + 网上的一些博客，其实部署还是比较容易的，但是我在实践过程中还是踩了一些坑，特此记录一下。

我使用的是 Helm 的部署方式，会将其部署到 k8s 上。

这篇文章目前只是简单记录了一下搭建的过程和 harbor 的简单使用，并没用对 harbor 进行详细的探索。

部署

使用 Helm 部署，首先需要准备一份 values.yaml，这个 yaml 的模版可以从 这个 repo 里获取（你也可以直接复制我下面提供的），然后需要根据自己的需求，对这个模版进行修改。

这里我修改了这几部分：

  • type：用于指定 harbor 的 service 的类型，我修改成了 nodePort。

  • commonName：如果开启了 TLS 则必填，且必须和 externalURL 的域名部分相同，即 https:// 后面的，但是端口号可以忽略，比如如果 externalURL 为 https://core.harbor.domain:30003 ，那么 commonName 需要指定为 core.harbor.domain。如果 commonName 和 externalURL 域名部分不同，会导致一些操作（比如 docker 的 login 和 push ）出现证书验证失败的问题。

    如果你和我一样不小心掉到坑里了，把 commonName 配置的和 externalURL 域名部分不一致，导致出现 x509: certificate is valid for harbor-cert, not core.harbor.domain 错误，那么恭喜你，可以重头再来了。。。开玩笑的，其实只要更新一下 values.yaml，然后执行 helm upgrade harbor harbor/harbor -f values.yaml -n harbor 更新一下 release 就可以了。

  • externalURL：指定 harbor 的访问域名，这里要注意的是，如果指定的是 nodePort 类型的，那么必须要加上端口号，比如 https://core.harbor.domain:30003 这样。

tls 这里我为了方便就直接让 Harbor 自动创建证书文件了，没有像 其他博客 一样创建自定义证书，所以 certSource: auto 这里保持原样即可，无需修改。

还有一个需要注意的地方是 harborAdminPassword: "Harbor12345" 这里，指定的是 harbor admin 用户的登录密码，这里要不要改就看自己的需求了，如果只是测试一下，感觉没啥改的必要。

还有如果指定的 service type 是 nodeport 的话，这个 yaml 会把 nodePort 的端口写死，比如默认 http 对应的 nodePort 端口是 30002，而 https 是 30003。这个要不要改也是看自己的需求了，我这里是没改。

使用 Docker 推送镜像

首先执行 docker login 命令，登录到 harbor：

如果这一步遇到了 x509: certificate signed by unknown authority 错误，那么你需要在 /etc/docker/daemon.json 里加入一条 "insecure-registries": ["core.harbor.domain:30003"]，然后执行 sudo systemctl restart docker 重启 docker 让其生效。

如果执行后输出 Login Succeeded ，那么恭喜你，登录成功了。
